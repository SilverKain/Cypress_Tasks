Waterfall Model	Последовательная модель разработки с четкими фазами: Анализ → Проектирование → Разработка → Тестирование → Внедрение → Поддержка. QA: Тестирование происходит после завершения разработки. Практические задачи: Создание детального Test Plan на фазе анализа, подготовка тест-кейсов до начала разработки.

Agile Testing	Итеративный подход к тестированию в рамках гибкой методологии. Принципы: Раннее и непрерывное тестирование, тесное сотрудничество с командой, быстрая обратная связь. Роль QA: Участие во всех церемониях Scrum, написание acceptance criteria, exploratory testing. Практические задачи: Daily testing, sprint retrospectives, continuous integration testing.

V-Model	Модель разработки в виде буквы V, где каждой фазе разработки соответствует фаза тестирования. Левая часть: Requirements → System Design → Architecture → Module Design. Правая часть: Unit Testing → Integration Testing → System Testing → Acceptance Testing. QA роль: Планирование тестов параллельно с проектированием.

Scrum Framework	Agile-фреймворк с ролями (Product Owner, Scrum Master, Development Team), событиями (Sprint, Daily Standup, Sprint Review, Retrospective) и артефактами (Product Backlog, Sprint Backlog, Increment). QA роль: Участие как Development Team member, Definition of Done, Sprint Testing. Практические задачи: Sprint planning estimation, backlog grooming.

Spiral Model	Итеративная модель с акцентом на управление рисками. 4 фазы: Определение целей → Анализ рисков → Разработка и тестирование → Планирование следующей итерации. QA роль: Risk-based testing, создание прототипов для тестирования. Практические задачи: Риск-анализ функций, приоритизация тестирования по рискам.

Kanban	Система управления потоком работ с визуализацией через доску Kanban. Принципы: Визуализация работы, ограничение WIP, управление потоком, явные политики. QA роль: Управление тестовыми задачами через доску, контроль качества на каждом этапе. Практические задачи: WIP лимиты для testing колонок, метрики cycle time.

DevOps Culture	Культура объединения разработки и операций для ускорения поставки качественного ПО. Принципы: Автоматизация, непрерывная интеграция/доставка, мониторинг, быстрая обратная связь. QA роль: Shift-left testing, test automation, quality gates в CI/CD. Практические задачи: Pipeline testing, infrastructure testing, monitoring качества.

Incremental Model	Модель, где продукт разрабатывается по частям (инкрементам), каждый инкремент добавляет функциональность. QA роль: Тестирование каждого инкремента, регрессионное тестирование предыдущих инкрементов. Практические задачи: Incremental test strategy, baseline testing, integration testing между инкрементами.

Extreme Programming (XP)	Agile-методология с акцентом на технических практиках. Практики: Pair Programming, TDD, Continuous Integration, Simple Design, Refactoring. QA роль: Участие в TDD процессе, customer testing, acceptance testing. Практические задачи: Writing user stories с acceptance criteria, exploratory testing sessions.

Lean Software Development	Методология, основанная на принципах бережливого производства. 7 принципов: Устранение потерь, усиление обучения, поздние решения, быстрая поставка, расширение полномочий команды, встроенное качество, видение целого. QA роль: Встроенное качество, value stream mapping. Практические задачи: Waste identification в testing процессах.

Test Planning Process	Процесс планирования тестирования включает анализ требований, определение стратегии тестирования, оценку ресурсов и создание расписания. Артефакты: Test Plan, Test Strategy, Test Schedule. QA роль: Ведущая роль в планировании. Практические задачи: Risk analysis, test estimation, resource allocation.

Requirements Analysis Phase	Фаза анализа требований в SDLC. QA активности: Участие в requirements review, создание acceptance criteria, трассируемость требований. Артефакты: Requirements Traceability Matrix, Test Basis. Практические задачи: Анализ testability требований, выявление gaps в требованиях.

Design Phase Testing	Тестирование на фазе проектирования. QA активности: Design review, test design based on technical specifications, создание test architecture. Принципы: Early defect detection, test design in parallel. Практические задачи: Создание test cases из design documents, API testing strategy.

Implementation Phase	Фаза разработки кода. QA роль: Code review participation, unit test support, early integration testing. Принципы: Continuous testing, fast feedback. Практические задачи: Static code analysis, code coverage monitoring, build verification tests.

Testing Phase	Dedicated testing phase в традиционных моделях. Активности: Test execution, defect reporting, test reporting. QA роль: Основная роль в планировании и выполнении тестов. Практические задачи: Test execution management, defect triage, test progress reporting.

Deployment Phase	Фаза внедрения системы в production. QA роль: Production readiness testing, smoke testing after deployment, deployment verification. Принципы: Zero-downtime deployment, rollback procedures. Практические задачи: Deployment testing checklist, production monitoring setup.

Maintenance Phase	Фаза поддержки после release. QA активности: Regression testing для bug fixes, testing новых features, performance monitoring. Роль QA: Support team collaboration, maintenance testing strategy. Практические задачи: Hotfix testing procedures, maintenance test automation.

Test Strategy Document	Высокоуровневый документ, определяющий общий подход к тестированию проекта. Содержание: Test objectives, test approach, test deliverables, risks and mitigation. QA роль: Создание и поддержание актуальности. Практические задачи: Stakeholder alignment, strategy communication, approach justification.

Test Plan Document	Детальный план тестирования для конкретного релиза или итерации. Содержание: Test items, features to test, testing approach, pass/fail criteria, test deliverables, testing schedule. QA роль: Test Manager ответственность. Практические задачи: Detailed planning, resource coordination, timeline management.

Entry Criteria	Условия, которые должны быть выполнены перед началом тестирования. Примеры: Готовность test environment, completion of code development, availability test data. QA роль: Определение и контроль соблюдения. Практические задачи: Criteria definition, readiness assessment, go/no-go decisions.

Exit Criteria	Условия завершения тестирования. Примеры: Test coverage achievement, defect resolution rates, performance benchmarks met. QA роль: Определение критериев и оценка их достижения. Практические задачи: Criteria measurement, completion assessment, sign-off recommendations.

Defect Life Cycle	Жизненный цикл дефекта от обнаружения до закрытия. Статусы: New → Open → Assigned → Fixed → Retest → Verified → Closed. QA роль: Управление жизненным циклом, verification и closure. Практические задачи: Defect triage, retest execution, closure verification.

Test Case Management	Процесс управления тест-кейсами от создания до execution. Активности: Test case creation, review, maintenance, execution tracking. QA роль: Test case ownership и management. Практические задачи: Test case repository organization, version control, execution tracking.

Test Execution Process	Процесс выполнения тестов. Этапы: Test environment setup, test data preparation, test execution, result recording, defect reporting. QA роль: Process facilitation и execution. Практические задачи: Execution planning, result documentation, issue tracking.

Test Reporting	Процесс создания отчетов о тестировании. Типы: Daily test reports, test execution reports, test summary reports, defect reports. QA роль: Report creation и stakeholder communication. Практические задачи: Metrics collection, trend analysis, stakeholder communication.

Risk-Based Testing	Подход к тестированию, основанный на анализе рисков. Принципы: Risk identification, risk assessment, test prioritization based on risk. QA роль: Risk analysis facilitation, risk-based test planning. Практические задачи: Risk workshops, probability/impact assessment, mitigation strategies.

Test Environment Management	Управление тестовыми средами. Активности: Environment provisioning, configuration management, data management, environment monitoring. QA роль: Requirements definition, environment validation. Практические задачи: Environment setup procedures, configuration baseline, data refresh strategies.

Configuration Management	Управление версиями и конфигурациями в контексте тестирования. Принципы: Version control, baseline management, change control. QA роль: Test artifacts versioning, environment configuration tracking. Практические задачи: Test code repository management, configuration documentation.

Test Data Management	Управление тестовыми данными. Активности: Test data identification, creation, maintenance, security. QA роль: Data requirements definition, data privacy compliance. Практические задачи: Data generation strategies, sensitive data masking, data cleanup procedures.

Continuous Integration Testing	Тестирование в рамках CI процесса. Принципы: Automated testing, fast feedback, early defect detection. QA роль: Test automation strategy, pipeline quality gates. Практические задачи: CI test suite design, build quality checks, automated reporting.

Continuous Delivery	Практика автоматической доставки ПО в production-ready состояние. QA роль: Deployment pipeline testing, quality gates definition, production readiness validation. Принципы: Automated deployment, quality assurance at each stage. Практические задачи: Pipeline testing strategy, automated acceptance tests.

Sprint Planning	Scrum церемония планирования спринта. QA роль: Story acceptance criteria review, testing task estimation, Definition of Done clarification. Активности: Backlog item analysis, capacity planning, commitment to sprint goal. Практические задачи: Test effort estimation, dependency identification.

Daily Standup	Ежедневная синхронизация команды в Scrum. QA участие: Progress reporting, impediments raising, coordination with developers. Формат: What did I accomplish yesterday? What will I do today? What obstacles are impeding my progress? Практические задачи: Testing progress updates, blocker identification.

Sprint Review	Демонстрация результатов спринта stakeholders. QA роль: Feature demonstration support, quality metrics presentation, feedback collection. Цель: Inspect increment, adapt product backlog. Практические задачи: Quality assessment presentation, defect metrics review, stakeholder feedback incorporation.

Sprint Retrospective	Команда анализирует процесс и планирует улучшения. QA участие: Process improvement suggestions, testing efficiency analysis, team collaboration assessment. Формат: What went well? What could be improved? What will we commit to improve? Практические задачи: Testing process optimization, tool evaluation.

Product Backlog Grooming	Процесс уточнения и приоритизации backlog items. QA роль: Acceptance criteria refinement, testability assessment, complexity estimation participation. Цель: Ready backlog for upcoming sprints. Практические задачи: Story splitting for testability, acceptance criteria review, dependency analysis.

Definition of Done	Общее понимание команды о том, когда work item считается завершенным. QA влияние: Testing criteria inclusion, quality standards definition, acceptance criteria integration. Принципы: Transparency, quality assurance, shared understanding. Практические задачи: DoD criteria definition, compliance monitoring.

User Story Testing	Тестирование пользовательских историй. Подход: Acceptance criteria validation, scenario-based testing, user journey testing. QA роль: Story testing strategy, acceptance test creation. Практические задачи: BDD scenarios creation, user acceptance testing, story completion validation.

Acceptance Criteria	Условия, определяющие когда пользовательская история готова. Формат: Given-When-Then, checklist, narrative. QA роль: Criteria review и refinement, testability validation. Практические задачи: Criteria quality assessment, test case derivation, completeness verification.

Epic Testing	Тестирование больших пользовательских историй (эпиков). Подход: Epic breakdown testing, integration testing across stories, end-to-end scenario validation. QA роль: Epic test strategy, cross-story testing coordination. Практические задачи: Epic acceptance criteria, integration test planning.

Feature Testing	Тестирование функций продукта. Подход: Feature-based test design, feature integration testing, feature acceptance validation. QA роль: Feature test planning, comprehensive feature validation. Практические задачи: Feature test suite design, feature sign-off criteria, cross-feature impact analysis.

Release Planning	Планирование релизов продукта. QA роль: Release criteria definition, testing timeline planning, quality gates establishment. Активности: Scope definition, risk assessment, resource planning. Практические задачи: Release test strategy, quality milestone definition, regression testing planning.

Version Control for Testing	Управление версиями тестовых артефактов. Принципы: Test code versioning, test data versioning, environment configuration versioning. QA роль: Test repository management, version control best practices. Практические задачи: Branching strategies for tests, merge conflict resolution, release tagging.

Test Automation Strategy	Стратегия автоматизации тестирования. Компоненты: Automation scope, tool selection, maintenance strategy, ROI calculation. QA роль: Strategy development, implementation oversight. Практические задачи: Automation pyramid design, tool evaluation, automation roadmap creation.

Shift-Left Testing	Перенос тестирования на более ранние этапы SDLC. Принципы: Early testing, prevention over detection, developer collaboration. QA роль: Early involvement, requirements review, design validation. Практические задачи: Requirements testability analysis, early test design, prevention strategies.

Quality Gates	Контрольные точки качества в процессе разработки. Примеры: Code coverage gates, defect density thresholds, performance benchmarks. QA роль: Quality criteria definition, gate monitoring, go/no-go decisions. Практические задачи: Criteria establishment, automated gate implementation, compliance monitoring.

Test Metrics	Метрики для измерения эффективности тестирования. Типы: Process metrics (test coverage, defect density), progress metrics (test execution rate), quality metrics (defect escape rate). QA роль: Metrics collection, analysis, reporting. Практические задачи: KPI definition, dashboard creation, trend analysis.

Defect Triage	Процесс приоритизации и назначения дефектов. Активности: Defect classification, priority assignment, resource allocation. QA роль: Triage facilitation, severity assessment. Практические задачи: Triage meetings, severity/priority matrix, assignment criteria.

Test Coverage	Измерение степени покрытия тестированием. Типы: Requirements coverage, code coverage, risk coverage. QA роль: Coverage planning, measurement, gap analysis. Практические задачи: Coverage metrics definition, coverage tools setup, gap identification and mitigation.

Exploratory Testing	Исследовательское тестирование с одновременным изучением, проектированием и выполнением тестов. Принципы: Learning, test design, execution в real-time. QA роль: Session planning, execution, findings documentation. Практические задачи: Session charter creation, note-taking techniques, session reporting.

Test Design Techniques	Методы проектирования тестов. Категории: Black-box (equivalence partitioning, boundary value analysis), white-box (statement coverage), experience-based (exploratory testing). QA роль: Technique selection, application, training. Практические задачи: Technique selection criteria, test case optimization.

Regression Testing	Тестирование для подтверждения, что изменения не нарушили существующую функциональность. Стратегии: Full regression, selective regression, prioritized regression. QA роль: Regression suite design, execution planning. Практические задачи: Test case selection, automation prioritization, execution optimization.

Smoke Testing	Базовое тестирование основной функциональности для проверки стабильности build. Цель: Build verification, deployment validation. QA роль: Smoke test suite design, execution criteria. Практические задачи: Critical path identification, automated smoke tests, fast feedback mechanisms.

Sanity Testing	Узконаправленное тестирование подмножества функциональности после minor changes. Отличие от smoke: более глубокое, но узкое тестирование. QA роль: Sanity test scope definition, execution. Практические задачи: Change impact analysis, focused test execution, quick validation.

Integration Testing	Тестирование взаимодействия между компонентами или системами. Уровни: Component integration, system integration, end-to-end integration. QA роль: Integration test design, interface testing, data flow validation. Практические задачи: Integration points identification, test environment setup, interface validation.

System Testing	Тестирование полной интегрированной системы. Типы: Functional system testing, non-functional system testing, business process testing. QA роль: System-level test planning, end-to-end scenario execution. Практические задачи: System test environment setup, end-to-end test design, system acceptance criteria.

User Acceptance Testing	Тестирование системы end users для подтверждения соответствия бизнес-требованиям. Типы: Business UAT, operational UAT, contract UAT. QA роль: UAT facilitation, user training, criteria validation. Практические задачи: UAT planning, user coordination, acceptance criteria verification.

Performance Testing Strategy	Стратегия тестирования производительности. Типы: Load testing, stress testing, volume testing, spike testing. QA роль: Performance requirements analysis, test design, results interpretation. Практические задачи: Performance baseline establishment, bottleneck identification, scalability testing.

Security Testing Integration	Интеграция тестирования безопасности в SDLC. Подходы: Static analysis, dynamic analysis, penetration testing. QA роль: Security testing coordination, vulnerability assessment. Практические задачи: Security test planning, vulnerability scanning, secure coding validation.

API Testing Strategy	Стратегия тестирования API. Аспекты: Functional testing, performance testing, security testing, contract testing. QA роль: API test design, automation strategy, integration testing. Практические задачи: API documentation validation, contract testing implementation, automation framework setup.

Database Testing	Тестирование базы данных и связанных операций. Аспекты: Data integrity, data validity, performance, triggers testing. QA роль: Database test design, data validation, performance monitoring. Практические задачи: SQL query validation, data migration testing, backup/recovery testing.

Mobile Testing Approach	Подход к тестированию мобильных приложений. Особенности: Device fragmentation, network conditions, battery usage, app store guidelines. QA роль: Mobile test strategy, device coverage planning. Практические задачи: Device matrix creation, mobile automation, app store compliance.

Cross-browser Testing	Тестирование веб-приложений в различных браузерах. Аспекты: Functional compatibility, UI consistency, performance variations. QA роль: Browser matrix definition, compatibility validation. Практические задачи: Browser support matrix, automated cross-browser testing, responsive design validation.

Accessibility Testing	Тестирование доступности приложений для пользователей с ограниченными возможностями. Стандарты: WCAG guidelines, Section 508, ADA compliance. QA роль: Accessibility requirements validation, compliance testing. Практические задачи: Screen reader testing, keyboard navigation, color contrast validation.

Usability Testing	Тестирование удобства использования и пользовательского опыта. Методы: User interviews, usability sessions, A/B testing. QA роль: Usability test planning, user feedback collection. Практические задачи: User journey testing, UI/UX validation, user feedback analysis.

Test Environment Strategy	Стратегия управления тестовыми средами. Компоненты: Environment types, provisioning strategy, data management, access control. QA роль: Environment requirements definition, strategy implementation. Практические задачи: Environment topology design, automation of environment setup, environment monitoring.

Test Data Strategy	Стратегия управления тестовыми данными. Аспекты: Data generation, data masking, data refresh, data privacy. QA роль: Data requirements definition, data quality assurance. Практические задачи: Test data identification, synthetic data generation, data privacy compliance.

Incident Management	Процесс управления инцидентами в production. QA роль: Post-incident analysis, root cause analysis participation, prevention measures. Процесс: Incident detection, response, resolution, post-mortem. Практические задачи: Incident response procedures, RCA facilitation, prevention strategy development.

Change Management	Управление изменениями в контексте QA. Аспекты: Impact assessment, testing strategy adaptation, regression analysis. QA роль: Change impact analysis, testing scope adjustment. Практические задачи: Change request review, impact assessment, test plan updates.

Quality Assurance vs Quality Control	QA фокусируется на процессах и предотвращении дефектов, QC - на продукте и обнаружении дефектов. QA: Process improvement, prevention, proactive. QC: Product inspection, detection, reactive. Практические задачи: Process auditing vs product testing, prevention strategies vs detection methods.

Test Process Improvement	Улучшение процессов тестирования. Подходы: TPI (Test Process Improvement), TMM (Test Maturity Model), assessments. QA роль: Process analysis, improvement initiative leadership. Практические задачи: Current state assessment, improvement roadmap creation, implementation monitoring.

Root Cause Analysis	Анализ первопричин дефектов и проблем процессов. Методы: 5 Whys, Fishbone diagram, Fault Tree Analysis. QA роль: RCA facilitation, pattern analysis, prevention recommendations. Практические задачи: Defect pattern analysis, process gap identification, preventive action planning.

Test Review Process	Процесс review тестовых артефактов. Типы: Test plan review, test case review, test automation code review. QA роль: Review facilitation, quality criteria enforcement. Практические задачи: Review checklist creation, review meeting facilitation, action item tracking.

Estimation Techniques	Методы оценки тестирования. Подходы: Expert judgment, historical data analysis, function point analysis, use case point method. QA роль: Estimation accuracy, effort planning. Практические задачи: Effort estimation models, accuracy tracking, estimation refinement.

Test Tool Evaluation	Оценка и выбор инструментов тестирования. Критерии: Functionality, usability, integration capabilities, cost, support. QA роль: Requirements definition, tool comparison, recommendation. Практические задачи: Tool selection criteria, POC execution, vendor evaluation.

Test Team Management	Управление командой тестировщиков. Аспекты: Team structure, skill development, task allocation, performance management. QA роль: Team leadership, resource planning, skill gap analysis. Практические задачи: Team capacity planning, skill matrix maintenance, performance reviews.

Stakeholder Communication	Коммуникация с заинтересованными сторонами проекта. Аудитории: Business stakeholders, development team, management, end users. QA роль: Quality status reporting, risk communication, requirement clarification. Практические задачи: Status reporting, quality metrics presentation, risk escalation.

Test Deliverables	Артефакты, производимые в процессе тестирования. Примеры: Test plans, test cases, test reports, defect reports, test summary reports. QA роль: Deliverable creation, quality assurance, stakeholder distribution. Практические задачи: Template creation, content quality control, distribution management.

Quality Standards	Стандарты качества ПО. Примеры: ISO/IEC 25010 (SQuaRE), ISO 9001, CMMI. QA роль: Standards compliance, implementation guidance, audit support. Практические задачи: Standards gap analysis, compliance planning, audit preparation.

Test Closure Activities	Активности завершения тестирования. Включают: Test summary reporting, lesson learned sessions, test environment cleanup, test artifact archiving. QA роль: Closure process facilitation, knowledge transfer. Практические задачи: Final report preparation, retrospective facilitation, artifact handover.

Agile Testing Quadrants	Модель, классифицирующая типы тестирования в Agile. Q1: Unit tests (Technology facing, Support team). Q2: Functional tests (Business facing, Support team). Q3: Exploratory testing (Business facing, Critique product). Q4: Performance tests (Technology facing, Critique product). QA роль: Testing strategy alignment.

Testing in SAFe	Тестирование в масштабированном Agile (SAFe). Уровни: Team level testing, Program level testing, Portfolio level testing. QA роль: Multi-level testing coordination, integration testing across teams. Практические задачи: System team collaboration, integration testing in Program Increments.

Feature Driven Development Testing	Тестирование в FDD. Процессы: Develop Overall Model, Build Feature List, Plan by Feature, Design by Feature, Build by Feature. QA роль: Feature verification, integration testing. Практические задачи: Feature-based test planning, incremental testing, feature completion validation.

Crystal Methodology Testing	Тестирование в Crystal методологии. Принципы: Frequent delivery, close communication, reflective improvement, personal safety. QA роль: Adaptive testing strategy, reflection participation. Практические задачи: Testing strategy adaptation, team reflection facilitation, continuous improvement.

Lean Startup Testing	Тестирование в контексте Lean Startup. Концепции: Build-Measure-Learn cycle, MVP testing, validated learning. QA роль: Hypothesis testing, metrics definition, learning validation. Практические задачи: MVP test strategy, A/B testing support, metric-driven testing.

Design Thinking in QA	Применение дизайн-мышления в QA. Этапы: Empathize (понимание пользователей), Define (определение проблем), Ideate (генерация идей), Prototype (прототипирование), Test (тестирование). QA роль: User-centric testing, empathy building. Практические задачи: User journey mapping, persona-based testing.
